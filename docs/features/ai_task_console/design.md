# design.md — AI 开发任务控制台

## 目标

这个 starter 的首页不是营销页，而是任务控制台。

它负责：

1. 选择当前工程
2. 选择执行模式
3. 填写任务卡
4. 生成 `Execution Prompt`
5. 生成结构化 `Task Card`

## 控制台结构

主区固定为：

1. 工程
2. 执行模式
3. 任务卡
4. 固定协议
5. Execution Prompt
6. 高级输出

## 设计原则

1. 固定规则只读
2. 当前任务只填变量
3. 主输出必须是 `Execution Prompt`
4. 高级输出默认收起
5. 页面可直接在 `file://` 场景打开

## 初始化原则

starter 的初始化不能依赖用户手动改一串文件。

这一层现在要满足：

1. clone 完仓库后，先运行一个初始化脚本
2. 脚本统一收集项目名、展示名、默认对象、远端仓库等必要变量
3. 脚本一次性写回 `vibecoding.config.js`、`CLAUDE.md`、`AGENTS.md`、`README.md`
4. 初始化脚本完成写入后，必须自动执行控制台验证
5. 控制台自己的验证脚本必须基于当前配置运行，而不是写死 starter 默认值
6. 初始化完成后，用户应能直接进入控制台开始第一条真实任务
