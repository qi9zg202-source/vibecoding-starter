# design.md — AI 开发任务控制台

## 目标

这个 starter 的首页不是营销页，而是任务控制台。

它负责：

1. 选择当前工程
2. 选择执行模式
3. 填写任务卡
4. 生成 `Execution Prompt`
5. 生成结构化 `Task Card`

## 控制台结构

主区固定为：

1. 工程
2. 执行模式
3. 任务卡
4. 固定协议
5. Execution Prompt
6. 高级输出

## 设计原则

1. 固定规则只读
2. 当前任务只填变量
3. 主输出必须是 `Execution Prompt`
4. 高级输出默认收起
5. 页面可直接在 `file://` 场景打开
